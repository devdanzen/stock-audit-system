Key Patterns Used
  1. CardLayout — swaps content panels without opening new windows
  2. Header / Detail — Sales, Receiving, Recipes use this pattern
  3. Junction Tables — sales_detail, receiving_detail, recipe_detail resolve M-M relationships
  4. SQL Views — compute aggregates (SOH, top sellers) instead of storing them
  5. BCrypt — password hashing for USER table
  6. Transactions — header + detail INSERTs wrapped in setAutoCommit(false)
Dependencies
  • mysql-connector-j — JDBC driver
  • jbcrypt — password hashing
  • jfreechart — charts
  • jasperreports — reports (optional)
