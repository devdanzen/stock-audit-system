Build Phases — From Start to End

Project build order · priority sequence, not calendar time
01
Database & Schema
Foundation must come first.
Design 14 tables with FK relationships and 24 CHECK constraints, plus 4 SQL views (v_stock_on_hand, v_top_sellers, v_daily_sales, v_top_waste). Set up XAMPP + phpMyAdmin and import the schema.
Deliverables
inventory_schema_final.sql
Seed data (categories,
outlets, vendors, admin user)
02
Project Foundation
Set up the Java/Swing scaffolding.
Create the NetBeans project with the package layout (view / dao / model / util). Build shared utilities: DbConnection factory and PasswordUtil (BCrypt wrapper). Write Model POJOs for all 14 tables.
Deliverables
DbConnection.java
PasswordUtil.java
14 model POJOs
03
Authentication & Shell
Open the app, log in, see the empty home.
Build UserDAO, the LoginFrame, and the MainFrame with sidebar navigation + CardLayout content area. Wire role-based access so Staff / Manager / Administrator each see the right menu items.
Deliverables
LoginFrame.java
MainFrame.java
UserDAO.java
04
Master Data CRUDs
Populate the reference tables.
Build the five reusable CRUD panels: Items, Categories, Outlets, Vendors, Users. Each follows the same toolbar + form + JTable template — Items is built first as the reference, the rest copy its pattern.
Deliverables
5 panels · 4 DAOs
Shared CRUD template
(UserDAO already built in Phase 3)
05
Transactions
Stock starts moving.
Build the four transactional panels: Receiving, Sales, Movement, Recipes. Header + detail INSERTs wrapped in a single JDBC transaction (setAutoCommit(false), commit / rollback). Stock side-effects flow automatically through SQL views.
Deliverables
4 panels · 4 DAOs
Transaction wrapper pattern
06
Stock Operations
Inventory accountability.
Build Stock On Hand (reads v_stock_on_hand), Audit (editable JTable + live variance calculation), and End Balance (period snapshot with confirmation warning).
Deliverables
3 panels · 2 DAOs
Custom variance renderer
(Stock On Hand reads v_stock_on_hand view — no DAO)
07
Dashboard & Insights
Now that data flows, show it back.
Build the Dashboard (KPIs, recent transactions, low-stock alerts), Reports (filter + monospace preview, paginated), and Charts (4 JFreeCharts: top sellers, daily sales, sales by category, top waste).
Deliverables
Dashboard / Reports / Charts
JFreeChart integration
08
Polish & Defense
Make it presentable and shippable.
Add the About dialog, JOptionPane confirms for destructive actions, validation messages, look-and-feel tuning. Generate documentation, prepare the defense deck, run end-to-end testing across all 16 screens.
Deliverables
About dialog · CLAUDE.md
Defense slides · QA pass
Build principle: Each phase produces something runnable. By the end of Phase 3 you can log in. By Phase 5 the app already books real sales. Each later phase adds capability on top of a working system — never a half-broken intermediate state.
