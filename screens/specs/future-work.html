Future Work & Roadmap

Where the system could grow beyond the v1.0 final-project scope
New Features user-facing
  • Barcode / QR scanner — faster item lookup at Receiving and Audit
  • Mobile companion app — read-only Stock On Hand and quick Movement entry for staff on the floor
  • Bilingual UI — Bahasa Indonesia ↔ English toggle
  • Low-stock notifications — push alerts when items hit critical threshold
  • Recipe-driven sales deduction — selling a finished dish auto-deducts ingredients from raw stock
Technical Improvements platform
  • REST API layer — expose data to other systems and future clients
  • Change-log / audit trail table — who edited what, and when
  • Automated database backups + scripted restore workflow
  • Performance hardening — composite indexes on transaction tables for multi-year history
  • Unit + integration test coverage for the DAO layer
Integrations external systems
  • POS sync — auto-import sales from the cashier system at end-of-day
  • Vendor portal — vendors submit delivery slips and invoices directly
  • Accounting export — CSV + direct connect for the company's bookkeeping software
  • BI hand-off — read-only views consumable by Looker / Power BI / Tableau
UX & Polish presentation
  • Printable invoice / receipt slip with QR code for quick lookup
  • Customizable dashboard widgets per user role
  • Keyboard shortcuts for the most-used actions (new sale, save, search)
  • Dark mode and in-app theme switcher
  • Comprehensive in-app help & quick-search across all screens
Stretch goals: Multi-currency support · Multi-tenant architecture (other restaurant chains) · Forecasting models for reorder points · Supplier price-history tracking · Cloud deployment with role-based replicas per outlet.
