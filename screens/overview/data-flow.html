How the Data Connects

One action on one screen — many screens updated automatically. Plain English overview.
What happens when you record a SALE?
1 · User action
Cashier opens the Sales screen
Picks 3 items from the dropdown and enters quantities.
→
2 · System saves
Clicks "Save Transaction"
System creates one invoice record plus a separate line item record for each item sold.
→
3 · Stock effect
Stock goes DOWN
Quantity for each sold item is deducted automatically — no manual update.
→
4 · Now visible in
  • Dashboard — today's sales updates
  • Stock On Hand — lower quantities
  • Charts — Top Sellers refreshes
  • Reports — sales report shows it
What happens when goods ARRIVE from a vendor?
1 · User action
Staff opens the Receiving screen
Picks a vendor from the dropdown and lists what was delivered.
→
2 · System saves
Clicks "Save Receipt"
System creates one receipt record plus a separate line item record for each delivered item.
→
3 · Stock effect
Stock goes UP
Quantity for each received item is added automatically — no manual update.
→
4 · Now visible in
  • Dashboard — recent transactions list
  • Stock On Hand — higher quantities
  • Reports — receiving log report
How does Stock On Hand know the current quantity?
Starting Balance from last month-end
← from End Balance snapshot
+
All Receivings this period
← from Receiving screen
+
All Transfers IN
← from Movement screen
−
All Sales
← from Sales screen
−
All Waste
← from Movement screen
−
All Transfers OUT
← from Movement screen
−
All Consumption
← from Movement screen
=
Current Stock On Hand
← shown in Stock On Hand panel
The system recalculates this automatically every time you open the Stock On Hand screen — no manual updates needed.
