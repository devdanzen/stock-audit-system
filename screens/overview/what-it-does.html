The Business Problem We're Solving
📊
Restaurant chains lose money when stock isn't tracked. Items expire, get stolen, or run out at the wrong time.
📋
Manual Excel tracking is error-prone. Numbers don't match between sheets. Nobody knows the real stock level.
💡
Our app replaces messy spreadsheets with one organized system where every transaction updates inventory automatically.
A Day in the Life of the Restaurant Manager
09:00
Morning Check
Manager opens the Dashboard. Sees 3 items running low. Decides what to order today.
Screen used: Dashboard, Stock On Hand
→
10:30
New Delivery Arrives
Vendor delivers fish. Manager records the receipt — what came in, from whom, how much. Stock goes UP automatically.
Screen used: Receiving
→
12:15
Lunch Rush
Customers order food. Each invoice is recorded. Stock goes DOWN automatically for ingredients used.
Screen used: Sales
→
14:00
Kitchen Waste
Chef accidentally burns a batch of edamame. Manager records the waste to track losses.
Screen used: Movement (WASTE)
→
18:00
Transfer to Sister Outlet
SHLMP runs out of salmon but SHMTA has extra. Manager records the transfer between outlets.
Screen used: Movement (OUT)
→
22:00
End of Day
Manager runs an audit, walks the storage, counts items. App compares real stock vs system stock. Any difference is flagged for investigation.
Screen used: Audit
Who Uses Which Screens
Restaurant Staff (role = Staff)
  • Records daily sales (Sales)
  • Records goods receipt when a vendor arrives (Receiving)
  • Records waste, transfers, consumption (Movement)
Outlet Manager (role = Manager)
  • Reviews the Dashboard each morning
  • Approves audits at end of period (Audit)
  • Generates reports for headquarters (Reports)
  • Reviews sales trends (Charts)
Head Office / Administrator (role = Administrator)
  • Manages master data (Items, Categories, Vendors, Outlets, Users)
  • Reviews charts across all outlets (Charts)
  • Closes monthly periods (End Balance)
  • Manages user access (Users)
